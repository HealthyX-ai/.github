## Severity Levels

| Severity | Definition | Response Target |
|----------|------------|----------------|
| **P1 — Critical** | PROD is down, data integrity at risk, or PHI may be exposed | 2 hours |
| **P2 — High** | Major feature broken with no workaround | 4 business hours |
| **P3 — Medium** | Feature degraded, workaround exists | 1–2 business days |
| **P4 — Low** | Minor issue, cosmetic, or enhancement request | Next sprint |

## Filing an Issue

Use the [bug report template](../../issues/new?template=bug_report.md). Include:

- Which environment (POC / TST / SUP / PROD)
- Steps to reproduce
- What you expected vs. what happened
- Screenshots or log output (scrubbed of PHI)

**Do not include PHI, patient data, or credentials in any GitHub issue.**

---

## Escalation Path

1. File a GitHub issue with the correct severity label
2. Contact the implementation lead directly for P1 or P2 issues
3. For PROD-critical incidents: contact HealthyX on-call directly
